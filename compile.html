FOSScraft Compilation Guide

A comprehensive guide to building FOSScraft from source code on Linux and Windows platforms.

Last Updated: December 2025 | Applies to: Latest (develop branch)

Overview

FOSScraft is built using CMake and requires a modern C++ compiler, Qt6, and various dependencies. This guide provides step-by-step instructions for compiling FOSScraft on your system.

What You'll Need

  • CMake 3.21 or higher
  • Qt 6.x (6.4+ recommended)
  • C++ Compiler (GCC 11+, Clang 12+, or MSVC 2019+)
  • Git for version control
  • Java Development Kit (JDK) for some build components

System Requirements

Linux

Recommended Specifications

  • CPU: Multi-core (4+ cores)
  • RAM: 4GB minimum (8GB+ recommended)
  • Disk: 5-10GB for source and build
  • OS: Modern Linux with glibc 2.31+

Supported Distributions

  • Arch Linux / Manjaro
  • Fedora / RHEL / CentOS
  • Debian / Ubuntu 20.04+
  • openSUSE
  • Gentoo
  • Alpine Linux

Windows

Recommended Specifications

  • CPU: Multi-core (4+ cores)
  • RAM: 4GB minimum (8GB+ recommended)
  • Disk: 10-15GB for source and build
  • OS: Windows 10 / 11 (64-bit)
  • Admin: Admin access required

Visual Studio Requirements

  • Visual Studio 2019 or 2022
  • Community Edition (free)
  • C++ workload with MSVC
  • CMake support

Linux Compilation

Step 1: Install Dependencies

Select your distribution below to see the appropriate commands:

sudo pacman -S base-devel cmake qt6-base qt6-svg qt6-imageformats git jdk-openjdk
sudo dnf groupinstall "Development Tools" "Development Libraries" sudo dnf install cmake qt6-qtbase qt6-qtsvg git java-latest-openjdk-devel
sudo apt-get update sudo apt-get install build-essential cmake git openjdk-17-jdk-headless sudo apt-get install qt6-base-dev qt6-svg-dev libgl1-mesa-dev
For Ubuntu 22.04 or newer, Qt6 packages are available in the standard repository. Older versions may need Qt6 from the official Qt repository.
sudo zypper install gcc gcc-c++ cmake git java-17-openjdk-devel sudo zypper install libqt6-qtbase-devel libqt6-qtsvg-devel libqt6-qtimageformats-devel

Step 2: Clone the Repository

git clone https://github.com/FOSScraft/FOSScraft.git cd FOSScraft git submodule update --init --recursive

Step 3: Create Build Directory

mkdir build cd build

Step 4: Configure with CMake

cmake -DCMAKE_BUILD_TYPE=Release ..

Optional CMake Flags:

# Build with debug information cmake -DCMAKE_BUILD_TYPE=Debug .. # Specify custom installation prefix cmake -DCMAKE_INSTALL_PREFIX=/opt/fosscraft .. # Enable LTO for smaller binary cmake -DCMAKE_BUILD_TYPE=Release -DENABLE_LTO=ON .. # Set platform identifier cmake -DCMAKE_BUILD_TYPE=Release -DLauncher_BUILD_PLATFORM=archlinux ..

Platform Identifiers:

Distribution Identifier
Arch Linux archlinux
Fedora fedora
Debian debian
Ubuntu ubuntu
openSUSE opensuse
Gentoo gentoo
NixOS nixpkgs

Step 5: Build

# Single-threaded build make # Multi-threaded build (recommended - replace 4 with your CPU core count) make -j4 # Alternative: using ninja (faster) ninja

Step 6: Install

# Install to system sudo make install # Or install to custom directory make install

Step 7: Run

# If installed to /usr/local fosscraft # If installed to custom location /opt/fosscraft/bin/fosscraft # Or run directly from build directory ./launcher/FOSScraft

Configuration Directory:

FOSScraft will automatically create configuration files in:

~/.config/FOSScraft/

Windows Compilation

Step 1: Install Visual Studio Build Tools

  1. Download Visual Studio Community Edition
  2. Run the installer
  3. Select "Desktop development with C++"
  4. Ensure "MSVC v143 or later" and "CMake tools for Windows" are checked
  5. Complete installation (~10GB disk space)

Step 2: Install Qt6 for Windows

Option A: Using Qt Online Installer (Recommended)

  1. Download Qt Online Installer
  2. Create/login to Qt Account
  3. Select Components:
    • Qt 6.x → MSVC 2022 64-bit (or MSVC 2019 if using VS2019)
    • Developer and Designer Tools → CMake, Ninja
  4. Note the installation path (e.g., C:\Qt\6.x\msvc2022_64)

Option B: Using vcpkg

git clone https://github.com/Microsoft/vcpkg.git cd vcpkg .\bootstrap-vcpkg.bat .\vcpkg install qt6-base:x64-windows qt6-svg:x64-windows qt6-imageformats:x64-windows

Step 3: Clone Repository

git clone https://github.com/FOSScraft/FOSScraft.git cd FOSScraft git submodule update --init --recursive

Step 4: Create Build Directory

mkdir build cd build

Step 5: Configure with CMake

Open "Developer Command Prompt for Visual Studio" and run:

REM With Qt from Online Installer cmake -G "Visual Studio 17 2022" -A x64 ^ -DCMAKE_PREFIX_PATH="C:\Qt\6.x\msvc2022_64" ^ -DCMAKE_BUILD_TYPE=Release ..

With vcpkg:

cmake -G "Visual Studio 17 2022" -A x64 ^ -DCMAKE_TOOLCHAIN_FILE="C:\path\to\vcpkg\scripts\buildsystems\vcpkg.cmake" ^ -DCMAKE_BUILD_TYPE=Release ..

CMake Flags Explanation:

  • -G "Visual Studio 17 2022" - Use Visual Studio 2022 (use 16 for VS2019)
  • -A x64 - 64-bit architecture
  • -DCMAKE_PREFIX_PATH - Path to Qt installation
  • -DCMAKE_BUILD_TYPE=Release - Optimized build (use Debug for debugging)

Step 6: Build

Option A: Using CMake

cmake --build . --config Release --parallel 4

Option B: Using Visual Studio

REM Open in Visual Studio start FOSScraft.sln REM Or build from command line msbuild FOSScraft.sln /p:Configuration=Release /m:4

Step 7: Run

The executable is located at:

.\launcher\Release\FOSScraft.exe REM or .\launcher\Release\fosscraft.exe

Creating a Portable Build

REM From build directory cmake --install . --config Release --prefix ".\FOSScraft-portable"

The vcpkg or Qt deployment tool will handle runtime library copying.

Build Configuration

Customizing API Keys and Features

Edit CMakeLists.txt before building to customize:

# Meta server URL set(Launcher_META_URL "https://meta.fosscraft.org/v1/") # News RSS feed set(Launcher_NEWS_RSS_URL "https://fosscraft.org/news/feed.xml") # Bug tracker set(Launcher_BUG_TRACKER_URL "https://github.com/FOSScraft/FOSScraft/issues") # Community links set(Launcher_DISCORD_URL "https://fosscraft.org/discord") set(Launcher_MATRIX_URL "https://fosscraft.org/matrix") set(Launcher_SUBREDDIT_URL "https://fosscraft.org/reddit")

Platform-Specific Build Options

# Set for your distribution set(Launcher_BUILD_PLATFORM "archlinux") # Enable/disable features set(ENABLE_LTO ON) # Link Time Optimization set(CMAKE_BUILD_TYPE "Release") # or "Debug"

Troubleshooting

Linux Issues

"Qt6 not found" error

# Install qt6 development files for your distribution # Or manually specify Qt path cmake -DCMAKE_PREFIX_PATH=/usr/lib/cmake/Qt6 ..

"java: command not found"

# Debian/Ubuntu sudo apt-get install default-jdk # Fedora sudo dnf install java-latest-openjdk-devel

Out of memory during build

# Reduce parallel build jobs make -j2

Missing development headers

# Ubuntu - install dev packages sudo apt-get install libgl1-mesa-dev libxkbcommon-x11-dev libxcb-* -y

Windows Issues

"CMake not found"

Ensure "CMake tools for Windows" is installed via Visual Studio Installer, or install CMake separately from cmake.org

"Qt6 not found"

  • Verify Qt installation path in CMake configuration
  • Ensure the correct MSVC version matches your Visual Studio version

"MSVC compiler not found"

Open "Developer Command Prompt for Visual Studio" (not regular PowerShell). This sets up the correct compiler environment.

Build fails with "fatal error"

cd build cmake --build . --config Release --clean-first cmake --build . --config Release --parallel 4

Antivirus interference

Some antivirus software may slow down builds significantly. Consider temporarily disabling during compilation.

General Issues

CMake configuration fails

rm -rf build/* cmake -DCMAKE_BUILD_TYPE=Release ..

Check compiler version

gcc --version clang --version

Check binary dependencies (Linux)

ldd ./launcher/FOSScraft

Check binary dependencies (Windows)

objdump -p ./launcher/FOSScraft.exe | grep NEEDED
